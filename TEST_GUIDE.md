# 테스트 가이드

## 프로그램 테스트하기

### 1. 기본 기능 테스트

#### 방법 1: 예시 파일로 테스트
```bash
# 1. 프로그램 실행
python capcut_motion.py

# 2. 파일 선택
examples/sample_draft.json 선택

# 3. 기본 설정으로 실행
모션 패턴: 기본 줌인
시작 스케일: 1.04
끝 스케일: 1.08~1.10
이동 범위: 0.05

# 4. "모션 적용하기" 클릭

# 5. 결과 확인
examples/sample_draft_motion.json 파일 생성 확인
```

#### 방법 2: 실제 CapCut 파일로 테스트

**⚠️ 주의: 반드시 테스트용 프로젝트로 시작하세요!**

```bash
# 1. CapCut에서 간단한 테스트 프로젝트 생성
# - 이미지 3~5개 추가
# - 각 이미지 2~3초 길이
# - 저장 후 CapCut 완전히 종료

# 2. draft_content.json 파일 위치 찾기
C:\Users\[사용자명]\AppData\Local\CapCut\User Data\Projects\[프로젝트명]\

# 3. 프로그램 실행 및 적용

# 4. CapCut 재실행하여 효과 확인
```

### 2. 각 모션 패턴 테스트

각 패턴을 하나씩 테스트해보세요:

#### 테스트 체크리스트
- [ ] 기본 줌인: 랜덤 줌 + 랜덤 이동 확인
- [ ] 줌인/줌아웃 랜덤: 두 방향 모두 나오는지 확인
- [ ] 좌→우 이동: 왼쪽에서 오른쪽으로 이동 확인
- [ ] 우→좌 이동: 오른쪽에서 왼쪽으로 이동 확인
- [ ] 상→하 이동: 위에서 아래로 이동 확인
- [ ] 하→상 이동: 아래에서 위로 이동 확인
- [ ] 중앙 줌: 위치 고정, 확대만 확인

### 3. 설정 값 테스트

#### 테스트 케이스 1: 미니멀
```

끝 스케일 최소: 1.08
끝 스케일 최대: 1.10
이동 범위: 0.05
```
→ **기대 결과**: 적당한 역동성

#### 테스트 케이스 3: 드라마틱
```
시작 스케일: 1.05
끝 스케일 최소: 1.10
끝 스케일 최대: 1.12
이동 범위: 0.07
```
→ **기대 결과**: 강한 움직임, 역동적

## 문제 발생 시 체크리스트

### GUI가 실행되지 않음
- [ ] Python 3.7 이상 설치 확인
- [ ] tkinter 설치 확인 (대부분 기본 포함)
- [ ] 오류 메시지 확인

### 모션이 적용되지 않음
- [ ] CapCut 완전히 종료되었는지 확인
- [ ] 파일 이름이 정확히 draft_content.json인지 확인
- [ ] 파일 교체를 올바르게 했는지 확인
- [ ] CapCut 버전 호환성 확인

### 백업 파일이 없음
- [ ] 백업 체크박스 활성화 확인
- [ ] backups/ 폴더 권한 확인
- [ ] 디스크 용량 확인

### EXE 빌드 실패
- [ ] PyInstaller 설치 확인: `pip show pyinstaller`
- [ ] Python 버전 확인 (3.7 이상)
- [ ] 관리자 권한으로 실행 시도
- [ ] 백신 프로그램 일시 비활성화

## 성공 확인 방법

### ✅ GUI 테스트 성공
- 프로그램 창이 정상적으로 열림
- 모든 버튼과 입력란이 작동
- 파일 선택 다이얼로그가 열림

### ✅ 모션 적용 성공
- `*_motion.json` 파일 생성됨
- CapCut에서 클립에 모션 효과 확인
- 타임라인에서 키프레임 보임

### ✅ 백업 성공
- `backups/` 폴더 생성됨
- 타임스탬프 파일명의 백업 파일 존재
- 백업 파일 내용이 원본과 동일

### ✅ EXE 빌드 성공
- `dist/CapCut_모션_자동화.exe` 생성됨
- EXE 더블클릭으로 실행됨
- Python 없는 PC에서도 실행 가능
